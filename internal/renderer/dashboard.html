Claude Code Statusline

Add to your ~/.claude/settings.json to show context usage in your statusline:

{
  "statusLine": {
    "type": "command",
    "command": "node /path/to/ctx-monitor/bin/statusline-bridge.mjs"
  }
}

Claude Code Hook

Add a PostToolUse hook to ~/.claude/settings.json for live tracking:

{
  "hooks": {
    "PostToolUse": [{
      "type": "command",
      "command": "node /path/to/ctx-monitor/bin/ctx-monitor.mjs --statusline --compact"
    }]
  }
}

Codex CLI tmux

Run Codex CLI alongside ctx-monitor in a tmux split:

# Launch Codex + ctx-monitor side by side
tmux new-session -d -s work "codex" \\
  split-window -h "node /path/to/ctx-monitor/bin/ctx-monitor.mjs --codex --watch 3" \\
  select-pane -t 0

SKILL Setup (Claude Code Plugin)

Create a ctx-monitor skill so Claude Code can show context usage on demand:

# Create skill directory
mkdir -p ~/.claude/skills/ctx-monitor

# Create SKILL.md
cat > ~/.claude/skills/ctx-monitor/SKILL.md << 'EOF'
---
name: ctx-monitor
description: Show context window composition breakdown
trigger: "context monitor|ctx monitor|show context|context usage"
allowed-tools: ["Bash"]
---

Run \`node /path/to/ctx-monitor/bin/ctx-monitor.mjs --claude --compact\`
to show a quick context composition summary, or
\`node /path/to/ctx-monitor/bin/ctx-monitor.mjs --claude --table\`
for detailed breakdown.
EOF
